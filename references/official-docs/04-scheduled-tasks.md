# Claude Code Scheduled Tasks (세션 내 반복 실행)

**원본 URL:** https://code.claude.com/docs/en/scheduled-tasks

---

## 개요

Claude Code의 Scheduled Tasks를 이용하면 세션 중에 프롬프트를 자동으로 반복 실행할 수 있습니다. 배포 완료 폴링, PR 감시, 빌드 상태 확인, 리마인더 설정 등에 유용합니다.

**주의**: 세션 내 스케줄링이므로, 세션을 닫으면 모든 작업이 삭제됩니다. 지속적 스케줄링은 Cloud 또는 Desktop 스케줄된 작업 또는 GitHub Actions 사용.

---

## 스케줄링 옵션 비교

| 항목 | Cloud | Desktop | `/loop` |
|------|-------|---------|---------|
| 실행 환경 | Anthropic 클라우드 | 로컬 머신 | 로컬 머신 |
| 머신 요구 | 불필요 | 필요 | 필요 |
| 세션 요구 | 불필요 | 불필요 | 필요 |
| 재시작 후 지속 | 예 | 예 | 아니오 |
| 로컬 파일 접근 | 아니오 | 예 | 예 |
| MCP 서버 | Connector | 설정 파일 + Connector | 세션에서 상속 |
| 권한 프롬프트 | 아니오 (자율) | 설정 가능 | 세션에서 상속 |
| 최소 간격 | 1시간 | 1분 | 1분 |

**추천**: Cloud는 무인 실행, Desktop은 로컬 접근 필요, `/loop`는 빠른 세션 내 폴링

## /loop Skill - 세션 내 반복 실행

### 기본 사용법

```bash
/loop 5m 배포가 완료되었는지 확인하고 결과 알려줘
```

Claude가 간격을 파싱하고, Cron 표현식으로 변환하고, 백그라운드 작업 스케줄링

### 간격 문법

선택적 간격. 앞/뒤 어느 곳에나 배치 가능:

| 형식 | 예시 | 해석 |
|------|------|------|
| 선행 토큰 | `/loop 30m 빌드 확인` | 30분마다 |
| 후행 `every` | `/loop 빌드 확인 매 2시간` | 2시간마다 |
| 간격 생략 | `/loop 빌드 확인` | 기본값 10분마다 |

지원 단위: `s` (초), `m` (분), `h` (시간), `d` (일)

초는 가장 가까운 분으로 올림. 정확히 나누어떨어지지 않는 간격은 가장 가까운 깔끔한 간격으로 올림 (Claude가 선택한 간격 확인)

### 다른 명령어 루핑

```bash
/loop 20m /review-pr 1234
```

작업이 실행될 때마다 `/review-pr 1234` 실행

## 일회성 리마인더

자연어로 일회성 작업 설정:

```bash
오후 3시에 release branch 푸시하도록 리마인드해줘
```

```bash
45분 후 통합 테스트 완료 여부 확인해줘
```

Claude가 특정 분, 시간에 Cron 표현식으로 정정하고 실행 시간 확인

## Scheduled Task 관리

자연어 요청:

```bash
어떤 스케줄된 작업을 가지고 있니?
```

```bash
deploy 체크 작업 취소해줘
```

또는 기본 도구 사용:

| 도구 | 용도 |
|------|------|
| `CronCreate` | 새 작업 스케줄. 5필드 Cron 표현식, 프롬프트, 반복/일회 여부 받음 |
| `CronList` | 모든 스케줄된 작업 목록 (ID, 스케줄, 프롬프트) |
| `CronDelete` | 작업 취소 (ID 전달) |

각 작업은 8글자 ID. 세션당 최대 50개 스케줄된 작업

## 작업 실행 메커니즘

스케줄러는 매초 체크해서 기한이 된 작업을 큐에 추가. 작업은 낮은 우선순위로 Claude가 응답 완료 후 실행. 사용자 입력 처리 중에는 대기.

모든 시간은 **로컬 타임존**으로 해석. Cron `0 9 * * *`는 UTC가 아닌 현지 9AM 의미.

### 지터 (Jitter)

모든 세션이 동시에 API 호출하는 것을 방지하기 위해 작은 결정론적 오프셋 추가:

- **반복 작업**: 기간의 10% 이내 지연, 최대 15분. 시간 단위 작업은 `:00`~`:06` 사이 실행
- **일회 작업**: 시간/분의 상/하단 실행 시 최대 90초 앞당김

오프셋은 작업 ID에서 파생되므로 동일 작업은 항상 같은 오프셋. 정확한 타이밍 필요 시 `:00`, `:30` 아닌 시간 (예: `3 9 * * *`) 선택하면 일회 지터 제외

### 7일 만료

반복 작업은 생성 후 7일 후 자동 만료. 마지막 실행 후 자신 삭제. 장기 반복 필요 시 Cloud/Desktop 스케줄된 작업이나 만료 전 재생성

## Cron 표현식 레퍼런스

표준 5필드 Cron: `minute hour day-of-month month day-of-week`

지원: 와일드카드 (`*`), 단일값 (`5`), 스텝 (`*/15`), 범위 (`1-5`), 쉼표 목록 (`1,15,30`)

| 예시 | 의미 |
|------|------|
| `*/5 * * * *` | 5분마다 |
| `0 * * * *` | 매시간 정각 |
| `7 * * * *` | 매시간 7분 |
| `0 9 * * *` | 매일 오전 9시 (로컬) |
| `0 9 * * 1-5` | 평일 오전 9시 (로컬) |
| `30 14 15 3 *` | 3월 15일 오후 2:30 |

요일: 0/7=일 ~ 6=토. `L`, `W`, `?`, `MON`, `JAN` 등 확장 문법 미지원.

## Scheduled Task 비활성화

환경 변수 설정:

```bash
export CLAUDE_CODE_DISABLE_CRON=1
```

Cron 도구와 `/loop`가 사용 불가능해지고, 스케줄된 작업 실행 중지.

---

## 세션 내 스케줄링 제한

- **머신 이상**: Claude Code 실행 중이고 유휴 상태일 때만 실행
- **Catch-up 없음**: 스케줄 시간에 Claude가 작업 중이면 한 번만 실행 (매 간격마다 아님)
- **재시작 후 삭제**: Claude Code 종료/세션 재시작 시 모든 작업 삭제

지속적 Cron 자동화는 Cloud/Desktop 스케줄된 작업 또는 GitHub Actions 사용

## 실전 예시

### 배포 상태 폴링

```bash
/loop 2m 배포 상태를 확인하고, 완료되면 알려줘
```

### PR 감시

```bash
/loop 10m /review-pr 123의 최신 변경사항 체크
```

### 주기적 체크

```bash
/loop 1h 린트 에러 재실행
```

### 일회 리마인더

```bash
내일 오전 10시에 release note 작성 리마인드
```
